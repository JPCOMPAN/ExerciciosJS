Por Que JavaScript no Back-end?

  • Popularidade do JavaScript no Front-end:
    • Essencial para dar interatividade às páginas web (validação de formulários, animações, transições).
    • Tradicionalmente usado do lado do cliente (o que vemos no navegador).
  • A Transição para o Back-end:
    • Inicialmente, JavaScript não foi desenvolvido para o lado do servidor; linguagens como Java, Python e Ruby dominavam.
    • Surgimento do Node.js: Uma plataforma que permite a execução de JavaScript do lado do servidor.
    • Unificação do Desenvolvimento Web: Possibilitou que programadores de front-end e back-end utilizassem uma linguagem só.
    • Benefícios: Facilita a colaboração, aumenta a eficiência, simplicidade e coesão do projeto.
  • JavaScript sem HTML: Aprenderemos a trabalhar com JavaScript e executá-lo sem precisar de uma estrutura de página HTML.

Características Essenciais da Linguagem JavaScript

  • Linguagem Interpretada:
    • Não é compilada como outras linguagens.
    • É interpretada por um interpretador, instrução por instrução, durante a execução.
    • Geralmente, linguagens interpretadas têm um tempo de execução mais rápido.
  • Linguagem Multiparadigma:
    • Permite organizar e montar programas de diversas formas.
    • Exemplos: Orientação a Objeto, Estruturada, entre outros.
    • É importante escolher um paradigma e manter a coesão no sistema.

Tipos de Dados em JavaScript

  • Numéricos: Representam valores numéricos.
  • String: Sequência de caracteres.
  • Booleanos: Valores lógicos (true ou false).
  • Array: Uma lista de valores.
  • Object: Uma coleção de pares chave-valor.
  • undefined: Variável declarada, mas sem valor.
  • null: Ausência intencional de valor.
// Exemplo de Tipos de Dados

let nome = "Alice";       // String
let idade = 30;            // Number
let aprovada = true;        // Boolean

let hobbies = ["leitura", "corrida", "código"]; // Array

let contato = {           // Object
  email: "alice@example.com",
  telefone: null      // null: telefone não fornecido
};

let sobrenome;             // undefined

console.log(typeof nome);     // "string"
console.log(typeof idade);    // "number"
console.log(typeof contato);   // "object"

Declaração de Variáveis e Constantes

  • Tipagem Dinâmica:
    • JavaScript não é uma linguagem fortemente tipada.
    • Você não precisa declarar o tipo da variável (ex: let string nome).
    • O JavaScript infere o tipo com base no valor atribuído (ex: let idade = 25 infere numérico).
    • Pode causar problemas em alguns momentos, mas há conversão de tipos.
  • Declaração de Variáveis:
    • let: Utilizado para declarar variáveis que podem ter seu valor alterado.
      • Possui escopo de bloco (a variável é reconhecida apenas dentro do bloco onde foi declarada).
      • Exemplo: let nome = "João";.
    • var: Embora funcione, é desaconselhável devido à complexidade de seu escopo e não será usado no curso.
  • Declaração de Constantes:
    • const: Utilizado para declarar valores que não vão mudar após a atribuição inicial.
      • Exemplo: const PI = 3.14;.
      • Impede que outras partes do programa alterem o valor.
      • Útil para valores como CPF ou IDs que não devem ser modificados.
// Exemplo de let e const

// Usando let para uma variável que pode ser alterada
let contador = 10;
console.log('Valor inicial:', contador); // Saída: Valor inicial: 10

contador = 20;
console.log('Valor alterado:', contador); // Saída: Valor alterado: 20

// Usando const para uma constante que não pode ser alterada
const TAXA_FIXA = 0.05;
console.log('Taxa fixa:', TAXA_FIXA); // Saída: Taxa fixa: 0.05

// A linha abaixo causaria um erro, pois não se pode reatribuir uma constante
// TAXA_FIXA = 0.10; // TypeError: Assignment to constant variable.

Operadores Aritméticos e de Atribuição

  • Operadores Aritméticos Básicos:
    • + (Adição)
    • - (Subtração)
    • * (Multiplicação)
    • / (Divisão)
    • % (Módulo): Devolve o resto de uma divisão inteira.
      • Útil para verificar se um número é par ou ímpar (resto 0 para par, diferente de 0 para ímpar).
  • Operador de Inversão de Sinal:
    • - (unário): Colocado na frente de uma variável, inverte seu sinal (positivo vira negativo, e vice-versa).
  • Operadores de Incremento e Decremento:
    • ++ (Incremento): Aumenta o valor da variável em 1.
    • -- (Decremento): Diminui o valor da variável em 1.
    • Pré-fixado (++variavel / --variavel): Incrementa/decrementa antes de usar o valor da variável.
    • Pós-fixado (variavel++ / variavel--): Usa o valor da variável e depois incrementa/decrementa.
  • Operador de Atribuição:
    • = (Igual): Atribui um valor a uma variável (ex: nome = "João"). 
// Exemplo de Operadores Aritméticos e de Atribuição

let a = 10;
let b = 5;

console.log('Soma:', a + b);          // Saída: Soma: 15
console.log('Subtração:', a - b);       // Saída: Subtração: 5
console.log('Multiplicação:', a * b); // Saída: Multiplicação: 50
console.log('Divisão:', a / b);         // Saída: Divisão: 2
console.log('Módulo:', a % b);          // Saída: Módulo: 0

let c = 20;
c += 10; // Equivalente a: c = c + 10
console.log('Atribuição com soma:', c); // Saída: Atribuição com soma: 30

let d = 5;
d++; // Incremento
console.log('Incremento:', d); // Saída: Incremento: 6

Operadores Lógicos e de Comparação

  • Operadores Lógicos (Retornam true ou false):
    • == (Igualdade): Verifica se os valores são iguais.
    • != (Diferença): Verifica se os valores são diferentes.
    • === (Identidade): Verifica se os valores são iguais E do mesmo tipo.
    • !== (Não Identidade): Verifica se os valores não são idênticos (ou não são iguais, ou não são do mesmo tipo, ou ambos).
  • Operadores de Comparação de Grandeza:
    • > (Maior que)
    • >= (Maior ou igual a)
    • < (Menor que)
    • <= (Menor ou igual a)
    • Podem ser usados em estruturas condicionais.
  • Operadores Lógicos (Booleanos):
    • && (AND): Retorna verdadeiro se ambas as expressões forem verdadeiras.
    • || (OR): Retorna verdadeiro se pelo menos uma das expressões for verdadeira.
    • ! (NOT): Inverte o valor lógico (verdadeiro vira falso, e vice-versa).
// Exemplo de Operadores Lógicos e de Comparação

let x = 5;
let y = '5';

console.log('Igualdade (==):', x == y);   // Saída: true (compara apenas o valor)
console.log('Identidade (===):', x === y); // Saída: false (compara valor e tipo)

let idade = 20;
let temCNH = true;

console.log('Pode dirigir? (AND):', idade > 18 && temCNH); // Saída: true
console.log('É maior de idade ou tem CNH? (OR):', idade > 18 || temCNH); // Saída: true
console.log('Não é maior de idade? (NOT):', !(idade > 18)); // Saída: false

Conversão de Tipos (Type Conversion)

  • Conversão Automática (Implícita):
    • Na maioria dos casos, JavaScript tenta converter tipos automaticamente.
    • Exemplo: Ao concatenar uma string com um número ("O número é " + 42), o número é convertido para string.
  • Conversão Explícita:
    • Podemos forçar a conversão usando funções específicas:
    • Number(): Converte um valor para o tipo numérico.
      • Exemplo: Number("123") transforma a string "123" no número 123.
    • String(): Converte um valor para o tipo string.
      • Alternativa: Método .toString() (ex: numero.toString()).
    • Boolean(): Converte um valor para o tipo booleano.
  • JavaScript oferece várias formas de realizar a mesma tarefa.
// Exemplo de Conversão de Tipos

let stringNumero = "123";
let numeroConvertido = Number(stringNumero); // Converte string para número
console.log(typeof numeroConvertido); // Saída: "number"

let numeroOriginal = 456;
let stringConvertida = String(numeroOriginal); // Converte número para string
console.log(typeof stringConvertida); // Saída: "string"

let valorFalso = 0;
let booleanoConvertido = Boolean(valorFalso); // Converte 0 para false
console.log(booleanoConvertido); // Saída: false

Estruturas de Decisão

  • Conceito Geral: As estruturas de decisão em JavaScript são muito similares a outras linguagens de programação.
  • if/else:
    • Permite executar um bloco de código de acordo com uma condição.
    • Exemplo básico de verificação de igualdade/diferença.
  • else if:
    • Permite fazer múltiplas verificações em sequência.
    • Apenas um else é permitido por bloco de if.
  • Operador Ternário:
    • Forma resumida de if/else em uma única linha/expressão.
    • Útil para inicializar variáveis com base em uma condição.
  • switch/case:
    • Presente em JavaScript, similar a Java e C++. (Não disponível em Python, por exemplo.)
    • Testa uma variável contra várias possibilidades (casos).
    • Usa break para sair da estrutura após um caso verdadeiro.
    • Permite usar strings nos casos (diferente de algumas outras linguagens).
    • default: Bloco executado se nenhum caso for compatível.

Estruturas de Repetição (Laços)

  • while (enquanto):
    • Executa um bloco de código enquanto uma condição lógica for verdadeira.
    • A condição é verificada antes da primeira execução do bloco.
    • Exemplo: Incremento de contador até uma condição ser falsa.
  • do/while (faça/enquanto):
    • A diferença principal: o bloco de código será executado pelo menos uma vez, antes da condição ser verificada.
  • for (para):
    • Usado quando se sabe quantas vezes um trecho de código precisa ser executado.
    • Estrutura: inicialização de variável, condição, e incremento (não necessariamente de 1 em 1).
    • Observação: Muitas vezes, for e while podem ser usados para o mesmo propósito, dependendo da avaliação do desenvolvedor.
  • for...of:
    • Utilizado para iterar sobre elementos iteráveis, como arrays.
    • Retorna o valor de cada elemento na iteração.
  • for...in:
    • Utilizado para iterar sobre as propriedades (atributos/chaves) de um objeto.
    • Retorna o nome da chave/propriedade em cada iteração, permitindo acessar o valor do objeto via objeto[chave].

Funções

  • Funções em JavaScript: Essenciais e existem várias formas de declará-las, focaremos nas mais comuns.
  • Declaração com function (Palavra Reservada):
    • Sintaxe: function nomeDaFuncao(parametro1, parametro2) { // bloco de código }.
    • Parâmetros não precisam ter seus tipos especificados em JavaScript (linguagem de tipagem dinâmica).
    • Exemplo: Função saudacao(nome) que imprime "Olá [nome]".
    • Como Chamar: saudacao("João").
  • Funções como Expressões (Atribuídas a Variáveis/Constantes):
    • Sintaxe: const nomeDaFuncao = function(parametro1) { // bloco de código }.
    • Pode usar const (para função inalterável) ou let (para função que pode ser reatribuída).
    • Comportamento da função é o mesmo; a diferença é a forma de declaração e o escopo/mutabilidade.
    • Como Chamar: nomeDaFuncao("Valor") (igual à forma anterior).

Objetos

  • Objetos em JavaScript: Também podem ser criados de diversas formas; veremos as duas mais comuns.
  • Declaração por Literal de Objeto (Mais Simples):
    • Sintaxe: let pessoa = { nome: "João", idade: 25, cumprimentar: function() { console.log("Olá!"); } }.
    • Usa chaves {} para definir propriedades (atributos) e métodos (funções).
    • Acesso a Propriedades: pessoa.nome (retorna "João").
    • Chamada de Métodos: pessoa.cumprimentar().
  • Declaração com Construtor de Objetos (Função Construtora):
    • Cria uma função que serve como "molde" para múltiplos objetos do mesmo tipo.
    • Sintaxe da Função Construtora: function Pessoa(nome, idade) { this.nome = nome; this.idade = idade; }.
    • Uso de this para referenciar a propriedade do objeto que está sendo criado, distinguindo-o do parâmetro da função.
    • Instanciando o Objeto: let pessoa1 = new Pessoa("Maria", 30);.
    • Essa forma é útil ao criar vários objetos do mesmo tipo.
  • Observação: Existem outras formas de criar objetos em JavaScript, e é recomendado pesquisar para conhecimento adicional.
